# tdd

Test-driven development for the DAG workflow. Guides you through red-green-refactor with test quality heuristics and failure triage. Complements DAG's doc-as-context philosophy by treating tests as living documentation of code behavior.

## Prerequisites

- A test framework and runner for your language (chosen during the session)
- Optional: pairs with the feature-workflow and pair-programming plugins

## Skills

| Skill | Invocation | What it does |
|-------|------------|--------------|
| tdd | `invoked by start-work or navigator` | Walks through red-green-refactor per acceptance criterion, with test quality checks and failure triage |

## Usage

The tdd skill is typically invoked automatically by start-work (when TDD mode is enabled) or by the navigator skill during pair programming. It can also be loaded directly by asking your AI tool to use it.
