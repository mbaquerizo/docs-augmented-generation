## 3. Format

```
<gitmoji> [scope?][:?] <message>

[body?]
```

The subject line (including gitmoji, scope, and message) shall not exceed 50 characters, and the optional body must be wrapped at 72 character lines. This format is required unless the commit message is auto-generated by git (i.e. merge, revert commits).

- gitmoji: emoji in unicode format expressing the type of change, from the list in [gitmoji.md](gitmoji.md).
- scope: Optional string that adds contextual information for the scope of the change. Omit when the message provides enough scope on its own. If including, append a colon before the message.
- message: A brief summary of the code changes, capitalized and in the imperative present tense.
- body: Optional free-form body after the message providing additional context or details about the code changes.
